# 文章页面右侧目录导航设计文档

**日期**: 2026-03-11  
**功能**: 右侧固定目录导航 (Right Side TOC)  
**状态**: 设计阶段

---

## 1. 功能概述

在文章页面右侧添加一个固定的目录导航栏，提供以下功能：
- 显示文章的所有标题层级 (H2-H4)
- 点击目录项平滑滚动到对应章节
- 高亮当前阅读的章节
- 响应式设计：桌面端显示在右侧，移动端隐藏

---

## 2. 设计目标

1. **提升阅读体验**: 长文章快速导航，无需滚动查找
2. **视觉清晰**: 简洁设计，不干扰主要内容阅读
3. **性能友好**: 纯 CSS + 少量 JS，无额外依赖
4. **响应式适配**: 完美适配各种屏幕尺寸

---

## 3. 架构设计

### 3.1 组件结构

```
layouts/
└── partials/
    └── right_toc.html          # 右侧目录模板

assets/
└── css/
    └── extended/
        └── right-toc.css       # 右侧目录样式

static/
└── js/
    └── right-toc.js            # 滚动监听和高亮逻辑
```

### 3.2 布局结构

```html
<body>
  <header>...</header>
  <main class="main">
    <div class="post-wrapper">
      <article class="post-single">
        <!-- 文章内容 -->
      </article>
      <aside class="right-toc">
        <!-- 右侧目录 -->
      </aside>
    </div>
  </main>
  <footer>...</footer>
</body>
```

### 3.3 CSS 布局方案

采用 CSS Grid 布局：
- 桌面端 (>= 1200px): 文章 70% + 右侧 TOC 250px
- 平板端 (768px - 1199px): 文章 100%，隐藏右侧 TOC
- 移动端 (< 768px): 文章 100%，隐藏右侧 TOC

---

## 4. 功能规格

### 4.1 目录生成

- 提取文章中的 H2、H3、H4 标题
- 保持标题层级缩进关系
- 生成带锚点的链接

### 4.2 滚动监听

- 使用 Intersection Observer API 监听标题进入视口
- 高亮当前最靠近视口顶部的章节
- 平滑滚动到点击的章节

### 4.3 样式规格

| 元素 | 规格 |
|------|------|
| 宽度 | 250px |
| 位置 | 固定定位，距离顶部 100px |
| 背景 | 透明或轻微背景色 |
| 字体大小 | 14px |
| 层级缩进 | 每级 12px |
| 当前项高亮 | 主题色左边框 + 加粗 |
| 最大高度 | 视口高度 - 200px，超出滚动 |

### 4.4 响应式断点

| 断点 | 行为 |
|------|------|
| >= 1200px | 显示右侧 TOC |
| < 1200px | 隐藏右侧 TOC，使用原有内联 TOC |

## 5. 实现细节

### 5.1 Hugo 模板逻辑

1. 检查文章是否有标题 (H2-H4)
2. 提取标题文本和 ID
3. 生成嵌套列表结构
4. 仅在 `ShowToc` 为 true 时显示

### 5.2 JavaScript 功能

1. **初始化**: 获取所有标题和目录项
2. **滚动监听**: 使用 IntersectionObserver 监听标题
3. **高亮更新**: 更新当前活动目录项的样式
4. **点击处理**: 平滑滚动到对应章节

### 5.3 CSS 样式

1. **布局**: Grid/Flex 布局
2. **定位**: sticky 定位保持可见
3. **滚动条**: 自定义滚动条样式
4. **过渡**: 高亮切换添加过渡动画

---

## 6. 与现有系统的集成

### 6.1 主题兼容性

- 复用 PaperMod 的 CSS 变量
- 保持视觉风格一致
- 不修改主题核心文件

### 6.2 配置集成

使用 Hugo 现有配置：
- `params.ShowToc`: 控制是否显示目录
- `params.TocOpen`: 目录默认展开状态
- `params.UseHugoToc`: 使用 Hugo 原生 TOC 生成

### 6.3 降级方案

- 无 JavaScript 时：目录仍可点击跳转，但无高亮功能
- 窄屏幕时：自动隐藏，使用原有内联 TOC

## 7. 性能考虑

1. **懒加载**: TOC 内容随页面一起加载，无额外请求
2. **轻量 JS**: 使用原生 API，无外部依赖
3. **CSS 优化**: 使用 CSS 变量，支持主题切换
4. **Intersection Observer**: 高效滚动监听，不触发重排

---

## 8. 无障碍考虑

1. **键盘导航**: 支持 Tab 键在目录项间切换
2. **ARIA 标签**: 添加适当的 aria-label
3. **焦点管理**: 点击后焦点转移到对应章节
4. **对比度**: 确保文字与背景对比度符合 WCAG 标准
