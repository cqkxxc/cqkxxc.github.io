# 文章页右侧目录导航设计（方案 A）

- 日期：2026-03-11
- 项目：Hugo + PaperMod 博客
- 目标：在文章页面右侧增加固定页内目录导航（桌面端显示，移动端隐藏）

## 1. 背景与目标

当前站点已启用 Hugo/PaperMod 的目录能力（`UseHugoToc`、`ShowToc`），但未提供稳定的右侧固定目录体验。需求是在不大幅侵入主题的前提下，为文章详情页增加可用、轻量、可维护的右侧导航。

## 2. 范围定义

### 2.1 在范围内

- 仅修改文章详情页（single post）展示结构
- 复用 Hugo 原生 TOC 数据（`.TableOfContents`）
- 桌面端显示固定右侧目录栏
- 移动端隐藏右侧目录栏

### 2.2 不在范围内

- 不改造列表页、首页、taxonomy 页
- 不引入复杂 JS 动态高亮逻辑（首版）
- 不改动 Live2D 现有注入逻辑

## 3. 方案对比与结论

- 方案 A（采用）：基于现有 TOC，做布局和样式增强
  - 优势：改动小、风险低、维护成本低
- 方案 B：自定义 TOC partial + 更深模板改造
  - 优势：可控性高；劣势：改动面更大
- 方案 C：前端 JS 动态生成目录
  - 优势：交互灵活；劣势：复杂度和维护成本高

**结论：采用方案 A。**

## 4. 设计细节

### 4.1 模板层

1. 新增 `layouts/_default/single.html` 覆盖（包裹式）
   - 保持主题主体结构兼容
   - 在页面容器中引入“正文区 + 右侧目录区”布局

2. 新增 `layouts/partials/right_toc.html`
   - 目录来源：`.TableOfContents`
   - 显示条件：仅文章页且目录非空时渲染

### 4.2 样式层

在 `assets/css/extended/` 增加 `right-toc.css`：

- 桌面端（建议 `min-width: 1200px`）启用两栏布局
- 右侧目录栏使用 `position: sticky`
- 目录容器支持纵向滚动
- 移动端隐藏目录栏

### 4.3 配置层

无需新增站点配置项，沿用现有：

- `UseHugoToc = true`
- `ShowToc = true`
- `TocOpen = true`

## 5. 边界条件

- 无 H2/H3 标题文章：目录为空时不显示目录栏
- 目录项过多：目录栏内部滚动，不影响正文
- 标题层级深：首版按 Hugo 输出原样展示，不做折叠

## 6. 验收标准（DoD）

1. 桌面端文章页右侧出现固定目录栏
2. 点击目录项可跳转到文章对应标题
3. 页面滚动时目录保持可见（sticky）
4. 手机端不显示右侧目录
5. 搜索、分类、标签、Live2D 功能不受影响

## 7. 测试策略

- 本地 `hugo server` 手工验证
- 用三类文章验证：
  - 标题丰富
  - 标题较少
  - 无目录
- 断点验证：
  - `>=1200px`：显示目录
  - `<1200px`：隐藏目录

## 8. 风险与控制

- 主题升级兼容风险：通过最小化覆盖和 partial 化降低风险
- 样式冲突风险：样式命名使用明确前缀，避免污染全局
- 空目录体验风险：通过渲染条件避免空壳 UI
